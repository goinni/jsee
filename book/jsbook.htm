1.1.1  arguments 属性

定义：arguments是对象的一个特殊属性。我们可以把它当做数组对象来理解

属性：

length, 获取arguments对象的长度。

callee, 引用当前正在执行的函数。

案例分析：

（一）获取方法参数及参数长度

window.onload = function(){

         //在页面加载完成时调用abc方法

         abc(10,20,30);

}

//方法定义

function abc(){

         //虽然abc方法 这里没有引用参数，但是arguments仍能捕获到

         arguments.length; //参数长度值为 3

         arguments[0];// 参数1: 值为10

         arguments[1];// 参数 2: 值为20

         arguments[2];// 参数 3: 值为30

}

（二） 获取自己的方法名

求1到n的自然数之和

function add(n){

         if(n == 1) {

                 return 1;

         }else{

                 return n + arguments.callee(n-1);//arguments.callee() 等同于add()

                 //[注]callee可以用在没有命名的函数 进行调用自身 如:(递归方法)

         }       

}

1.1.2  call 与 apply 用法

(一)  call 讲解

定义：它的第一个参数用作 this 的对象。其他参数都直接传递给函数自身。

案例分析：

function person1(arg0){

         alert(this.name)//值为：李四

         alert(arg0)// 值为：1234567

}

function person2(){

         this.name = '李四';

}

var p2 = new person2;

//person1中this 将被p2对象替代，并接受一个参数，且执行person1这个方法

person1.call(p2,'1234567');

//[注]参数可以为多个，如：object1.call(object2,arg1,arg2,arg3…,argN)

 

(二) apply 讲解

定义：它有两个参数，用作 this 的对象和要传递给函数的参数的数组

案例分析：

function person1(a,b,c){

         alert(this.name)//值为：李四

         alert("数组的第一个值 "+a)// 值为：1

         alert("数组的第二个值 "+b)// 值为：2

         alert("数组的第三个值 "+c)// 值为：3

}

function person2(){

         this.name = '李四';

}

var p2 = new person2;

person1.apply(p2,[1,2,3]) //与call方法的区别是第二参数为数组

1.1.4  void 运算符

void 运算符对任何值返回 undefined。该运算符通常用于避免输出不应该输出的值，例如，从 HTML 的 <a> 元素调用 JavaScript 函数时。要正确做到这一点，函数不能返回有效值，否则浏览器将清空页面，只显示函数的结果。例如：

<a href="javascript:window.open('about:blank')">Click me</a>

如果把这行代码放入 HTML 页面，点击其中的链接，即可看到屏幕上显示 "[object]"。

这是因为 window.open() 方法返回了新打开的窗口的引用。然后该对象将被转换成要显示的字符串。

要避免这种效果，可以用 void 运算符调用 window.open() 函数：

<a href="javascript:void(window.open('about:blank'))">Click me</a>

这使 window.open() 调用返回 undefined，它不是有效值，不会显示在浏览器窗口中。

提示：请记住，没有返回值的函数真正返回的都是 undefined。

1.1.10  indexOf() 和 lastIndexOf() 方法

定义：indexOf() 和 lastIndexOf() 返回的都是指定的子符串在另一个字符串中的位置，找不到则返回 -1。

区别：indexOf() 方法是从开头（位置 0）开始检索字符串，而 lastIndexOf() 方法则是从结尾开始检索。

1.1.11  localeCompare() 方法

定义：此方法对字符串进行排序，该方法有一个参数 是“要进行比较的字符串”。

返回的是下列三个值之一：

•     如果 String 对象按照字母顺序排在参数中的字符串之前，返回负数。

•     如果 String 对象等于参数中的字符串，返回 0

•     如果 String 对象按照字母顺序排在参数中的字符串之后，返回正数。

案例分析：

var oStringObject1 = new String("a");
var oStringObject2 = new String("b");
var iResult = oStringObject1.localeCompare(oStringObject2);//对两个字符串排序
if(iResult < 0) {
  alert(oStringObject1 + " 在 " + oStringObject2 + " 之前 ");
} else if (iResult > 0) {
  alert(oStringObject1 + " 在 " + oStringObject2 + " 之后 ");
} else {
  alert("两个字符串相等");
}

1.1.14  slice() 和 substring()

定义：这两种方法均可实现字符串截取，都可以接受一个域两个参数。

不同点：对于负数参数，slice() 会用字符串的长度加上参数，substring() 则将其作为 0 处理。

substring() 总把较小的数字作为起始位，较大的数字作为终止位。

案例分析：

var oStringObject = new String("hello world");
alert(oStringObject.slice(-3));             //输出 "rld" 等同于slice(字符串的长度+(-3))
alert(oStringObject.substring(-3));         //输出 "hello world" 
alert(oStringObject.slice(3, -4)); //输出 "lo w"
alert(oStringObject.substring(3, -4));      //输出 "hel" 等同于 substring(0,3)

1.1.15  逻辑 AND 运算符 由 && 表示

定义：逻辑 AND 运算的运算数可以是任何类型的，不止是 Boolean 值。

如果某个运算数不是原始的 Boolean 型值，逻辑 AND 运算并不一定返回 Boolean 值：

•     如果一个运算数是对象，另一个是 Boolean 值，返回该对象。

•     如果两个运算数都是对象，返回第二个对象。

•     如果某个运算数是 null，返回 null。

•     如果某个运算数是 NaN，返回 NaN。

•     如果某个运算数是 undefined，发生错误。

案例分析：

var x = 1; x>0 && alert("x大于0"); // 我们可以这样理解if(x>0) alert("x大于0")

1.1.16  逻辑 OR 运算符 由 || 表示

定义：与逻辑 AND 运算符相似，逻辑 OR 运算并不一定返回 Boolean 值

•     如果一个运算数是对象，并且该对象左边的运算数值均为 false，则返回该对象。

•     如果两个运算数都是对象，返回第一个对象。

•     如果最后一个运算数是 null，并且其他运算数值均为 false，则返回 null。

•     如果最后一个运算数是 NaN，并且其他运算数值均为 false，则返回 NaN。

•     如果某个运算数是 undefined，发生错误。

与逻辑 AND 运算符一样，逻辑 OR 运算也是简便运算。对于逻辑 OR 运算符来说，如果第一个运算数值为 true，就不再计算第二个运算数。

案例分析：

var x = 1 || 0; alert(x) //结果为1

1.1.17  charAt() 和 charCodeAt() 方法

定义： charAt() 方法返回的是包含指定位置处的字符的字符串；

charCodeAt() 方法得到的不是字符，而是字符代码；

案例分析：

var oStringObject = new String("hello world");
alert(oStringObject.charAt(1));    //输出 "e"
alert(oStringObject.charCodeAt(1)); //输出"101" 即小写字母 "e" 的字符代码

1.1.18  类型转换

(一)  toString()方法

定义：原始类型 Boolean 值、数字和字符串都有 toString() 方法，可以把它们的值转换成字符串。

案例分析：

//布尔值

alert(bFound.toString()); //输出 "false"
//数值

我们还可以将Number 类型用toString()方法转成二进制、八进制、十进制、十六进制。

var iNum = 10;
alert(iNum.toString(2));  //转二进制 输出 "1010"
alert(iNum.toString(8));  //转八进制 输出 "12"

(二)  parseInt ()方法

定义：

parseInt() 方法首先查看位置 0 处的字符，判断它是否是个有效数字；如果不是，该方法将返回 NaN，不再继续执行其他操作。但如果该字符是有效数字，该方法将查看位置 1 处的字符，进行同样的测试。这一过程将持续到发现非有效数字的字符为止，此时 parseInt() 将把该字符之前的字符串转换成数字。

字符串中包含的数字字面量会被正确转换为数字，比如 "0xA" 会被正确转换为数字 10。不过，字符串 "22.5" 将被转换成 22，因为对于整数来说，小数点是无效字符。

案例分析：

var iNum1 = parseInt("12345red");  //返回 12345
var iNum1 = parseInt("0xA");       //返回 10
var iNum1 = parseInt("56.9");      //返回 56
var iNum1 = parseInt("red");       //返回 NaN

parseInt() 方法还可以把二进制、八进制、十六进制或其他任何进制的字符串转换成整数。

以下输出结果均为10
alert(parseInt(1010,2));  //将二进制1010转为整数
alert(parseInt(12,8));    //将八进制12转为整数
alert(parseInt("a",16));  //将十六进制a转为整数
alert(parseInt(10,10));   //将十进制10转为整数

 

(三)  parseFloat ()方法

定义：

parseFloat() 方法从位置 0 开始查看每个字符，直到找到第一个非有效的字符为止，然后把该字符之前的字符串转换成整数。不过，对于这个方法来说，第一个小数点是有效字符。如果有两个小数点，第二个小数点将被看作无效的。 使用 parseFloat() 方法的另一不同之处在于，字符串必须以十进制形式表示浮点数，而不是用八进制或十六进制。该方法会忽略前导 0，所以八进制数 0102 将被解析为 102。对于十六进制数 0xA，该方法将返回 NaN，因为在浮点数中，x 不是有效字符。

案例分析：

var fNum1 = parseFloat("12345red");         //返回 12345
var fNum2 = parseFloat("0xA");              //返回 NaN
var fNum3 = parseFloat("11.2");             //返回 11.2
var fNum4 = parseFloat("11.22.33");         //返回 11.22
var fNum5 = parseFloat("0102");             //返回 102
var fNum6 = parseFloat("red");              //返回 NaN

1.1.19  闭包

对于闭包不做过多解释和讲解，大家只需要知道，方法中可以嵌套方法，内部方法可以调用外部方法的参数、变量、方法等。方法内部定义的变量名或方法名均不会与外部冲突。

  }
  return doAdd();
}

(function(){
alert("hello init")   
})();

         alert(o.a);

2.1.1  css样式获取

//获取element对象的CSS样式
//element dom对象
//css_property CSS样式属性 所有属性均按标准css样式书写,如：border-color 
function getCss(element, css_property) {
         var elm_style = element.currentStyle ? element.currentStyle : window
                          .getComputedStyle(element, null);
         // 定义返回值
         var xr = "";
         // 兼容处理
         if (elm_style.getPropertyValue) {
                 // 处理非IE
                 xr = elm_style.getPropertyValue(css_property);
         } else {
                 // 处理IE
                 // 将带有'-'样式属性转成驼峰写法 border-color <--> borderColor
                 css_property = css_property.replace(/(\-+)\w/g, function(s) {
                          // 将'-'字符后的字母大写
                          return s.toUpperCase();
                 }).replace("-", "");
                 xr = elm_style.getAttribute(css_property);
         }
         // 将可用的值返回
         return xr ? xr : elm_style[css_property] ? elm_style[css_property]
                          : element.style[css_property];
}
//使用案例
//获取id 为 divID元素的宽度
getCss(document.getElementById("divID"), "width");

         if (entity.setSelectionRange) {
                 entity.focus();
                 entity.setSelectionRange(pos, pos);
         } else if (entity.createTextRange) {
                 var range = entity.createTextRange();
                 range.collapse(true);

                          range.moveStart('character', pos - 1);

                          range.moveStart('character', pos);
                 }
                 range.select();
         }
}

function browserType() {
         if (navigator.userAgent.indexOf("MSIE") > 0) {
                 var b_version = navigator.appVersion;
                 var version = b_version.split(";");
                 var trim_Version = version[1].replace(/[ ]/g, "");
                 if (trim_Version == "MSIE6.0") {
                          return "IE6";
                 } else if (trim_Version == "MSIE7.0") {
                          return "IE7";
                 } else if (trim_Version == "MSIE8.0") {
                          return "IE8";
                 } else if (trim_Version == "MSIE9.0") {
                          return "IE9";
                 }
         } else if (navigator.userAgent.toLowerCase().indexOf("firefox") > 0) {
                 return "Firefox";
         } else if (navigator.userAgent.toLowerCase().indexOf("safari") > 0) {
                 return "Safari";
         } else if (navigator.userAgent.toLowerCase().indexOf("opera") > 0) {
                 return "Opera";

                 return "Gecko";

                 return "no";
         }
}
 

2.1.17  获取页面元素

//页面元素选择器，可以根据ID，类名，标签名获取元素，
// selecter 为 #id 或 .classname 或 标签名（div,ul,li等）
// 返回值为数组  获取返回值中指定对象 例如：see('#id')[0];
function see(selecter) {
         var myReg_Elm = /^[a-zA-z0-9]+$/; //标签名
         var myReg_Id = /^#[a-zA-z0-9\-]+$/; //ID名 #idName
         var myReg_Class = /^\.[a-zA-z0-9\-]+$/; //类名 .classname
         var elements = new Array();// 存放返回值
         if (myReg_Id.test(selecter)) {     //根据[ID 名]选择对象
                 selecter = selecter.substring(1);
                 elements.push(document.getElementById(selecter));
         } else if (myReg_Class.test(selecter)) {    //根据[类 名]选择对象
                 selecter = selecter.substring(1);
         var elems = document.all ? document.all : document.getElementsByTagName("*");
                 var cName, cNames;
                 for (var e = 0; e < elems.length; e++) {
                          cName = elems[e].className;
                          cNames = cName.split(" ");
                          for (var i = 0; i < cNames.length; i++) {
                                   if (cNames[i] == selecter) {
                                            elements[elements.length] = elems[e];
                                            continue;
                                   }
                          }
                 }
         } else if (myReg_Elm.test(selecter)) {
                 elements = document.getElementsByTagName(selecter); //根据[标签名]选择对象
         }
         return elements; //返回结果以数组的形式统一返回
}

2.1.18  延迟加载

//当页面元素都加载完成后再执行 fn 的方法
function complete(fn) {
         var completeindex = 1;
         if (typeof fn == "function") {
                 //第一次加载时，创建数组，用来缓存页面加载完成后执行的方法
                 if (!this.execFn) {
                          this.execFn = new Array(); 
                 }
                 this.execFn.push(fn); //将方法存入缓存器中
         }
         //页面加载已完成
         if (document.readyState == "complete") {
         //如果方法存在，则执行方法
                 if (this.execFn.length) {
                          //遍历每个将要执行的方法
                          for (var key in this.execFn) {
                                   this.execFn[key](); //执行方法
                                   this.execFn.splice(key, 1); //移除执行后的方法
                          }
                 }
                 completeindex = 0; //设置回调器不可用
         }
         if (completeindex) {
                 setTimeout(arguments.callee, 1); //反复自身调用，直到加载完成为止
         }
}
//使用案例：
complete(function () {
         alert("页面加载完成")
});

//获取cookie内容
// name cookie名
function getCookie(name) {
         var nameOfCookie = name + "=";
         var x = 0;
         while (x <= document.cookie.length) {
                 var y = (x + nameOfCookie.length);
                 if (document.cookie.substring(x, y) == nameOfCookie) {
                          if ((endOfCookie = document.cookie.indexOf(";", y)) == -1)
                                   endOfCookie = document.cookie.length;
                          return unescape(document.cookie.substring(y, endOfCookie));
                 }
                 x = document.cookie.indexOf(" ", x) + 1;
                 if (x == 0)
                          break;
         }
         return "";
}

2.1.21  事件委派

// o 被设置的对象,即DOM对象 如：document.getElementById(“divID”)
// type 事件类型 如：onclick,onchange,onkeydown等
// callback 回调方法 [注]回调方法中的this指向当前触发事的对象
function live(o, type, callback) {
         if (o.attachEvent) {      // IE
                 o.attachEvent(type, function (e) {
                          var e = e || event; // e 兼容处理事件
                          o.entity = e.target || e.srcElement;
                          if (typeof callback == "function") {
                                   callback.call(o.entity);
                          }
                 }, false);
         } else { // 非IE
         //将type前缀on去掉，例如：onclick ->click
                 o.addEventListener(type.substring(2), function (e) {
                          var e = e || event;
                          o.entity = e.target || e.srcElement;
                          if (typeof callback == "function") {
                                   callback.call(o.entity);
                          }
                 }, false);
         }
}
//使用案例：
//说明:ID为'uu'元素及其所有的子节点都将 添加单击事件

2.1.22  事件绑定

// o 被设置的对象,即DOM对象 如：document.getElementById(“divID”)
// type 事件类型 如：onclick,onchange,onkeydown等
// callback 回调方法 
function bind(o, type, callback) {
         if (o.attachEvent) {
                 // IE6+
                 o.attachEvent(type, callback, false);
         } else if (o.addEventListener) {
                 // chrome,firefox
                 o.addEventListener(type.substring(2), callback, false);
         } else {
                 o[type] = callback;
         }
}

2.1.23  事件解绑

// o 事件触发对象 
// type 事件类型  onclick,onmouseover,onkeyup等
// callback 事件处理函数 
// [注]解绑时，o和callback函数必须与绑定时的相同 
function unbind(o, type, callback) {
         if (o.removeEventListener) {
                 o.removeEventListener(type.substring(2), callback, false);
         } else if (o.detachEvent) {
                 o.detachEvent(type, callback);
         } else {
                 o[type] = null;
         }
}
